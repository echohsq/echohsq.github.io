hexo的部署与安装

之前的有学生优惠的腾讯云被我架设了一个WordPress,还没玩爽,因为没有备案而偃旗息鼓了,只好转战我大hexo了

安装篇:

环境配置:

1.node js(点击下载,建议LTS版本)
2.git
3.Cmd MarkDown
依次下载后就可以安装啦,如果你看过其他的博客的话,不用配置环境,按照软件的默认设置安装就好,唯一需要注意的是,node js和git软件的安装一定要记得把path项勾上(当然path原本是勾选的),不需要什么特别配置,一路next就好啦.
当node js和git安装好后,我们要先检查一下,是否可以调用,用win+r打开运行,输入cmd,通过两个命令来检查一下是否安装就位:

node -v
git –version

如果显示:

恭喜你,第一步成功了!

hexo的安装:

1.利用npm安装:
npm install hexo-cli -g
稍等片刻,我们的hexo便会安装成功了.
2.还记得我们的git么?
现在就是git出马的时候了,如果你还不知道的git版本控制,那么请先花10分钟阅读一下维基百科上关于git和github以及之后我们要部署的github pages的相关内容.
看完了?对么,那么让我们开始吧,如果没有记错的话,这时你的右键菜单中一定会多出两个这种东西

Git GUI here和Git Bash here

这就是我们需要的,GUI是图形窗口,图形窗口谁不会啊,所以我们选择打开bash,依次输入:cd d:/
mkdir hexo && cd hexo

然后执行:
hexo init 初始化hexo仓库
安装依赖包:
npm install
这时我们的hexo博客便已经搭建完成了.
执行hexo g 生成静态html的代码
再在bash中执行hexo s --debug就可在浏览器中输入http://localhost:4000来访问自己的博客啦!

部署篇:

关于github的账号注册以及创建repository问题,可参阅这里.本篇不再累述.
假设仓库已经准备好了,建议仓库地址使用ssh.具体可参阅stormZhang的公众号.
这时打开hexo目录下的_config.yml文件(使用sublime)滑到文件最底端,按照我的模板填写:

deploy:
type: git(注意,冒号之后有一个空格)
repo: 复制的仓库链接
branch: master

这时进入bash中,执行:
npm install hexo-deployer-git --save(防止出现不能识别git的错误)
hexo g
hexo d
这时浏览器中输入username.github.io即可访问你的博客了;可能你会觉的他的默认主题很丑,这是我们就需要在主题上花一番功夫了,这里写的很好.

填坑篇:

  • github的repository和theme部分我没有写,只是需要注意的是安装完hexo后的操作,全部需要在git bash中进行,千万不要在cmd中输入命令.
  • CNAME问题:
    当你有自己的域名时可以选择绑定到自己的域名,在你自己注册域名的地方设置两条CNAME跳转,分别是@匹配和www匹配,地址输入username.github.io,然后在hexo文件夹下的public文件夹下新建一个名为CNAME(注意大小写,没有后缀名),输入你自己的域名就好了.
  • public文件夹不更行问题,这个问题可能比较小众,一般都是由于你的主题配置文件_config.yml更改的格式不对,这时我们需要在hexo g时,注意输出的提示,观察哪里出现了问题,一般都是主题的插件没有安装,而开启了相应功能

    后续使用篇:

    /c/Users/h/AppData/Roaming/npm/hexo: line 12: node: command not found
    (更新于2017-02-04)
    在我配置好java的环境变量后,Git bash敲下命令hexo g，居然出现上述错误信息，然后发现输入任何有关hexo的命令都是同样的错误，我又试着敲了npm这个命令，发现也是出错的，于是我想估计是环境变量设置有问题，我在用户的环境变量PATH中添加了两个路径，分别是C:\Users\h\node_modules\hexo\bin和C:\software\nodejs，再加上原来就添加的路径C:\Users\h\AppData\Roaming\npm，发现就解决这个问题了，可以正常修改博文，然后发布网站。

不过，我也不清楚是否完全解决这个问题，还需要观察一段时间。
