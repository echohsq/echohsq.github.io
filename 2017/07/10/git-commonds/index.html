用Git的一些常用命令,原谅我脑子不好,看到的这些抄下来才放心!

Git 本地库命令:

在进行任何git操作时,都必须处在git仓库目录下,所以我们第一步要掌握的就是:

  • mkdir test(创建文件夹test)
  • cd test(切换到test目录)
  • touch a.md(新建a.md文件)

    接下来我们在该目录下创建git仓库:

    1.使用最多的一条命令:git status,git status命令为查看当前仓库的状态,使用非常频繁.
    2.接下来我们初始化第一个仓库:git init
    3.当我们初始化完成后,我们已经在文件夹中添加了一个a.md文件,所以我们要为该文件添加一个追踪(就是提交到我们的仓库中)该命令为git add 这时我们就将a.md写入到提交的缓存当中,当我们如果不想提交时可以使用git rm –cached来移除这个缓存.
    4.add进缓存之后就是提交到仓库了,这时我们使用git commit,具体操作为git commit -m’first commit’,commit是提交,而-m后输入的是我们提交时的提交信息.(有没有可以add和commit合体的操作?当然有,我们可以使用git commit -a -m’first commit’)
    5.git log可以查看commit记录.
    6.git branch团队合作利器:当我们git init后, 我们会默认有一个master目录,当我们想新建一个分支时,可以使用git branch a这个命令来新建一个a分支,再用git checkout a来切换到a分支,合起来使用的话就是git checkout -b a
    7.写完自己分支后的合并git merge,具体操作为先切换到master主目录,然后我们使用git merge a来将a目录来合并来,然后我们再使用git branch -d/D(强制删除)(git branch -d a)来删除分支a
    8.git tag创建版本号,具体操作为git tag v1.0,切换到某一版本时我们使用,git checkout v1.0

    我常用的一些alias:

  • git c = git commit
  • git s = git status
  • git co = git checkout
  • git br = git branch
  • git plm = git pull origin master
  • git psm = git push origin master
